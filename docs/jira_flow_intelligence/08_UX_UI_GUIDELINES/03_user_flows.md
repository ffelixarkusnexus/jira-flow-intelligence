# USER FLOWS

## First-Time User

1. Land on dashboard
2. Immediately see:
   - top insight
   - current bottleneck
3. No setup required

---

## Daily Usage

1. Open dashboard
2. Check:
   - alerts
   - bottleneck
3. Take action
4. Return next day

---

## Alert-Driven Flow

1. User receives alert
2. Opens dashboard
3. Sees context + insight
4. Investigates issue

---

## Design Principle

User should never need:
- training
- configuration
- interpretation of raw metrics
